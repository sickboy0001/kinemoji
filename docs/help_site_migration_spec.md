# チョイタメラボ：ヘルプドキュメント機能 移行仕様書（AI 実装指示書）

## 1. 概要

本仕様書は、「チョイタメラボ」の**ヘルプドキュメント機能**に特化し、これを別の Web システム（またはスタンドアロンのヘルプサイト）として実装・移行する際の指示書です。
ユーザーが利用方法（クイックスタート）、Markdown 記法、利用規約などを閲覧できる静的コンテンツ配信システムを構築します。

### 1.1 目的
- 既存の Markdown ファイルベースのヘルプシステムを、別のプラットフォームで再現する。
- 認証やデータベースを必要としない、軽量な「読み専」のヘルプサイトとして実装する。

### 1.2 対象機能
- ヘルプ記事の一覧表示（タブ/チップ形式）
- Markdown 記事の描画
- 記事間のナビゲーション
- レスポンシブデザイン（PC/スマホ対応）

---

## 2. コンテンツソース（データ元）

ヘルプ記事は、以下の Markdown ファイルをソースとして利用します。移行先システムでは、これらの内容を DB に取り込むか、静的ファイルとして読み込むようにしてください。

### 2.1 対象ファイル一覧
| ファイルパス | 記事スラッグ (slug) | 記事タイトル | 内容概要 |
|---|---|---|---|
| `src/contents/help/00_index.md` | `00_index` | 目次 | ヘルプ記事の一覧リンク |
| `src/contents/help/01_quickstart.md` | `01_quickstart` | クイックスタート | 基本的な利用手順（プロフィール、依頼、報告） |
| `src/contents/help/05_markdown.md` | `05_markdown` | Markdown ガイド | 記法説明（太字、見出し、テーブル、コードブロック） |
| `src/contents/help/90_terms.md` | `90_terms` | 利用規約 | 利用条件、免責事項、公開ガイドライン |

### 2.2 コンテンツ形式
- **フォーマット**: Markdown (`.md`)
- **対応機能**:
  - 標準 Markdown（見出し `#`、太字 `**`、リスト `-`、リンク `[text](url)`）
  - GFM 拡張（テーブル、タスクリスト、打ち消し線 `~~`）
  - コードブロック（言語指定付き：\`\`\`typescript, \`\`\`json など）
  - 引用ブロック（`> `）

## 3. 機能要件

### 3.1 ヘルプ記事表示ページ
単一のページで、スラッグ（URL パラメータ）に基づいて記事を表示します。

#### 3.1.1 URL 構造
- **基本形**: `/help/{slug}`
- **デフォルト**: `/help/00_index` または `/help`（index へリダイレクト）
- **例**:
  - `/help/01_quickstart` → クイックスタート記事を表示
  - `/help/05_markdown` → Markdown ガイドを表示

#### 3.1.2 ページ構成
1. **ヘッダー（Quick Links）**:
   - 利用可能な全記事へのリンクを「チップ（ボタン）」形式で横並び表示。
   - 現在表示中の記事は、アクティブ状態（色付き）で強調。
   - クリックで該当記事に遷移（ページ遷移またはスクロール）。
   - 例：`[00_index] [01_quickstart] [05_markdown] [90_terms]`

2. **メインコンテンツエリア**:
   - Markdown コンテンツをレンダリングした表示領域。
   - 装飾要件：現行の仕様に従うこと


3. **フッター（ナビゲーション）**:
   - 「一覧に戻る」または「前のページへ」ボタン。
   - 必要に応じて「次の記事へ」リンク。

### 3.2 Markdown レンダリング
- **ライブラリ推奨**: `react-markdown` (React 環境) または同等の Markdown パーサー。
- **必須対応書式**:
  - 見出し（h1〜h6）
  - 太字、斜体、打ち消し線
  - 箇条書き（リスト）、番号付きリスト
  - リンク（内部リンク `/help/xxx` 対応）
  - コードブロック（言語指定付き）
  - テーブル
  - 引用ブロック
- **セキュリティ**: XSS 対策のため、`dangerouslySetInnerHTML` の直接使用は避け、サニタイズ機能付きのライブラリを使用。

### 3.3 記事一覧取得
- 利用可能な記事スラッグのリストを取得する機能。
- **実装方法**:
  - **静的生成**: ビルド時にファイルシステムから一覧を生成（Next.js の `getStaticPaths` 等）。
  - **動的取得**: API エンドポイントから JSON 形式で取得（例：`/api/help-topics`）。
- **返却データ例**:
  ```json
  [
    { "slug": "00_index", "title": "目次" },
    { "slug": "01_quickstart", "title": "クイックスタート" },
    { "slug": "05_markdown", "title": "Markdown ガイド" },
    { "slug": "90_terms", "title": "利用規約" }
  ]
  ```

## 4. API 設計（必要場合）

移行先システムで API を用意する場合は、以下のエンドポイントを想定します。

### 4.1 記事取得 API
- **エンドポイント**: `GET /api/help/{slug}`
- **レスポンス**:
  ```json
  {
    "slug": "01_quickstart",
    "content": "# チョイタメラボ：クイックスタートガイド\n\n..."
  }
  ```
- **エラー**:
  - 404: 記事が見つからない場合
  - 500: サーバーエラー

### 4.2 記事一覧 API（オプション）
- **エンドポイント**: `GET /api/help`
- **レスポンス**:
  ```json
  [
    { "slug": "00_index", "title": "目次" },
    ...
  ]
  ```

## 5. UI/UX 詳細仕様

### 5.1 デザインシステム（参考）
既存システムからの移植を想定したスタイル指針：
- **メインカラー**: 濃い青（`#000080`）
- **フォント**: サンセリフ体（システムフォント）
- **レイアウト**:
  - コンテナ幅：最大 `1200px`（`max-w-5xl`）
  - 余白：上下 `2rem`、左右 `1rem`
  - カードデザイン：白い背景、薄い境界線、影（`shadow-xl`）
- **レスポンシブ**:
  - スマホ：チップをスクロール可能、余白を狭める
  - PC：チップを横並び、余白を広く


## 9. 移行時の注意点
1. **内部リンク**: Markdown 内のリンク（例：`[クイックスタート](01_quickstart.md)`）が、新しい URL 構造（`/help/01_quickstart`）に正しく遷移するか確認してください。
2. **画像**: 画像が含まれる場合（現状のヘルプにはなし）、画像パスの相対参照を絶対参照または適切なパスに変換してください。
3. **文字コード**: UTF-8 であることを確認し、日本語が正しく表示されるようにしてください。
